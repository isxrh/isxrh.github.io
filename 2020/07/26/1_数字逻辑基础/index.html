    1.1 模拟信号与数字信号

    自然界的物理量按其变化规律可分为两类：

    • 模拟量（analog）：数值和时间上都可以连续取值
    • 数字量（digital）：时间上离散，值域内只能取某些特定的值

    表示模拟量的信号成为模拟信号；表示数字量的信号成为数字信号。

    1.3 数制与数制间的转换

    十进制（Decimal）、二进制（Binary）、八进制（Octal）、十六进制（Hexadecimal）…

    1. 任意进制转换为十进制

    2.十进制转换为任意k进制

    将十进制数的整数部分除以k取余数，直到商为0，将余数逆序排列，得到k进制的整数部分；

    将十进制的小数部分乘以k，取出乘积的小数部分，剩余部分继续乘以k，直到满足精度要求为止，将乘积的整数部分顺序排列得到k进制的小数部分。

    3.二进制与八进制间的转换

    二进制八进制：以小数点为界，向左右两侧每3位分成一组（不够4位添0），每组转换为一位八进制数。

    例：(10111101.1101)2 =（010 111 101.110 100)2 = (275.64)8

    八进制二进制：每个数字拆成3位二进制数。

    例：(3641.256)8 = (11 110 100 001.010 101 11)2

    4.二进制与八进制间的转换

    二进制十六进制：以小数点为界，向左右两侧每4位分成一组（不够4位添0），每组转换为一位十六进制数。

    例：(101110110100100.1111011)2 = (0101 1101 1010 0100.1111 0110)2 = (5DA4.F6)16

    十六进制二进制：每个数字拆成4位二进制数。

    例：(B2E.57)16 = (1011 0010 1110.0101 0111)2

    1.4 代码

    1.二-十进制代码（BCD）

    用4位二进制数对一位十进制数进行编码，这种编码称为二-十进制编码（BCD, Binary Coded Decimal）

    最常用的8421BCD码：用00001001这10个二进制数， 依次代表09这10个十进制数的代码。

    【注】8421BCD码需先转换为10进制数，再转换为2进制数，不可直接转为二进制。

    2.格雷码（Gray Code）

    任意两个相邻码之间只有一位不同

    3.字符代码

    ASCII码: 美国信息交换标准码（American Standard Code for Information Interchange）

    1.5 二进制代码的表示法

    原码：本身

    反码：原码逐位求反，即1变为0，0变为1

    补码：

    • 利用反码求：反码的最低有效位加1
    • 利用原码求：二进制低位（包括小数部分）的第一个“1”（包含此1）右边保持不变，向左依次求反

    例：“11001” 原码：11001；反码：00110；补码：00111

    补码：2’s complement is also called Radix comlement ( 基数补码 )

    补码相对于基数互补。

    某n位数的补码是减去该数。

    反码：1’s complement is alse called Diminished radix complement ( 降基数补码 )

    反码相对于降基数互补。

    某n位数的补码是减去该数。
