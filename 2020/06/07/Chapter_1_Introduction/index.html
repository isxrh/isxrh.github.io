Chapter 1 Introduction

1.1 what is the Internet?

1.1.1 “nuts and bolts” view

  • millions of connected computing devices:
    • hosts = end systems
    • running network apps
  • communication links
    • fiber（光纤）, copper（铜线）, radio（无线电）, satellite（卫星）
    • transmission rate: bandwidth
  • Packet switches: forward packets (chunks of data)
    • routers and switches
  • End systems access the Internet through Internet Service Providers (ISPs)
  • End systems, packet switches, and other pieces of the Internet run protocols that control the sending and receiving of information within the Internet.
  • Internet ­standards are developed by the Internet Engineering Task Force (IETF) [IETF 2016]. The IETF standards documents are called requests for comments (RFCs)

1.1.2 A Services Description

  • Infrastructure that provides services to applications(distributed application):
    Web, VoIP, email, games, e-commerce, social nets, …
  • provides programming interface to apps(Application Programming Interface)

1.1.3 Whats a protocol?

A protocol defines the format and the order of messages exchanged between two or more communicating entities, as well as the actions taken on the transmission and/or receipt of a message or other event.
